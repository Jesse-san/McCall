This class, as stated by the doc strings, helps solve a dynamic program with transition matrices that can be described by Markov matrices.

class DP_Markov():
    """
    This class solves a dynamic program whose transition rule can be described 
    by a list of Markov matrices, one for each element of the control set U.
    The Bellman equation is:

           V(x_t) = max_{u_t \in U} (r(x_t,u_t) + beta EV(x_{t+1}|(x_t,u_t)))
    
    and the transition rule is a Markov matrix 
    
            M(u)
            
    that depends upon the control u \in U

    Parameters
    ----------
    beta : scalar
        the discount factor must be in [0,1)
    max_iter : int
        Maximum number of iterations for the value function solution
     : scalar
        Convergence parameter.
    X: list
        Names of the states
    U: list
        Names of the controls
    r: array
        function giving reward in state, control
    g: dictionary
        One Markov matrix for each control
    """

        self.max_iter = max_iter

        The control set has m elements

        """
        n = len(self.X)

        if np.array(self.r).shape != (n,m): print("The reward function is not well defined.")

    def _next_state_probs(self,x,u):
        """
        This function takes the state set, the control set, and the list of transition matrices
        It returns the relevant row of the transition matrix in state x, when control u is chosen.
    
        """
        return self.g[u][self.X.index(x)]
              
    def _next_value(self,value):
        """
        This function iterates on the value function using the Bellman equation.
    
        """
        v_next = []
        for state in self.X:
            bellman = []
            for control in self.U:
                probs = self._next_state_probs(state,control) 
                #self._next... because I need to call another function within the class
                bellman.append(self.r[self.X.index(state),self.U.index(control)]
                +self.beta*(value@probs))
            v_next.append(max(bellman))
            
        return v_next

    
    def solve(self):
        """
        Solves by repeated iteration on the value function.
        """
        value = np.zeros(len(self.X)) #start with zero value function
        check = 1.0 #check for convergence
        iter=0 #count iterations
        while check > self.:
            v1 = np.array(self._next_value(value))
            check = max(abs(value-v1))
            value = v1 #reset the value function
            iter += 1 #advance the counter
            if(iter > self.max_iter): #get out of Dodge, if we are in an infinite loop
                print("Algorithm did not converge")
                break
        return value

Solving a discrete version of the McCall model.¶

  • Suppose there are three job offers: 70,000, 85,000, and 75,000.
  • Unemployment insurance is worth 30,000
  • Discount factor, $\beta=0.9$

State Space:

$$ X = [w_{low} , w_{medium}, w_{high}] \ $$

Ideally, we want to maximize our wage or at least realize our reservation wages.

Controls:

$$M_o(Not Take) = \begin{bmatrix} 1/3 & 1/3 & 1/3\\ 1/3 & 1/3 & 1/3 \\ 1/3 & 1/3 & 1/3 \end{bmatrix}$$$$M_1(Take) = \begin{bmatrix} 1 & 0 & 0\\ 0 & 1 & 0 \\ 0 & 0 & 1 \end{bmatrix}$$

The Second Case¶

  • There are three job offers: 70,000, 85,000, and 75,000.
  • Unemployment insurance is worth 30,000
  • Discount factor, $\beta=0.9$

State Space: \ The state space is generally the same. $$ X = [w_{low} , w_{medium}, w_{high}] \ $$

Control Set: \ Here is where things differ a bit.

$$M_o(Not take) = \begin{bmatrix} 0.8 & 0.2 & 0\\ 0.5 & 0.5 & 0 \\ 0 & 0.7 & 0.3 \end{bmatrix}$$$$M_1(Take) = \begin{bmatrix} 1 & 0 & 0\\ 0 & 1 & 0 \\ 0 & 0 & 1 \end{bmatrix}$$

In the first scenario, the states had no bearing on the probabilities. The cutoff point was determined entirely by the initial wage offer and whether it is better than the continuation value (rejecting that wage and continuing to behave rationally, or essentially waiting for a better offer). In this first case, it is more appealing to turn down the lowest job offer of 70k because its value does not outweigh that of taking 30k in unemployment benefits and hoping for a better salary down the road. This makes sense intuitively, since taking a job early in life while forgoing further education or better opportunities may limit lifetime earnings.

In the second scenario, the states now have an impact on the transition probabilities. If an offer is not accepted, there is no longer have equal probabilities of getting a better offer next time. Conversely, the odds of receiving a better offer after turning down a job does not have a good outlook. The optimal policy, in this case, is to take any offer given.

One way I might conceptualize this scenario is that you get offers to work in different sectors which tend to have differing ranges of pay and differing chances for advancement. Let's say 70k is for a manual labor job, 85k is for government work(a bit unrealistic here), and 100k is business/finance. According to this model, once you are offered a job in manual labor or government work you cannot jump over to finance, but you can potentially do better than manual labor if you keep searching for a government job. Those finance jobs are hard to get, but having the requisite skills makes you desirable in the public sector. In this system however, all high school students learn their career aptitude early on and therefore pick the first job which gives them a credible offer.

def optimize(A, Beta, Q, k0, alpha):
    
    def prod(k):
        return A*k**alpha  #A is a special A and a is alpha

    def time_0(c):
        return prod(k0)-c[0]

    def time_1(c):
        k1 = prod(k0)-c[0]
        return prod(k1)-c[1]

    def time_2(c):
        k1 = prod(k0)-c[0]
        k2 = prod(k1)-c[1]
        return prod(k2)-c[2]
    
    def utility(c):
        u = 0
        T = len(c)
        if Q == 1:
            for i in range(T):
                u += Beta**i * np.log(c[i])
        else:
            for i in range(T):
                u += Beta**i * (c[i]**(1-Q)-1)/(1-Q)
        return u
    
    def objective(c):
        return -utility(c)
        

    def MRS(c):
        T = len(c)
        MRS =[]
        if Q == 1:
            for i in range(T):
                MRS.append(Beta**i/c[i])
        else:
            for i in range(T):
                MRS.append(Beta**i * c[i]**(-Q))
        return MRS
    
    def objective(c):
        return -utility(c)
    
    c0 = [1.0,1.0,1.0]
    bnds = ([0.1,200], [0.1,200], [0.1,200])
    
    cons = (
    {'type': 'ineq','fun': time_0},
    {'type': 'ineq','fun': time_1},
    {'type': 'eq', 'fun' : time_2}
        )
    
    u = opt.minimize(objective, c0, bounds = bnds, constraints = cons)
    c = u.x
    prices = MRS(c)
    interest_rates = [prices[0]/prices[1],prices[1]/prices[2]]
    
    if u['success']==True:
        return(interest_rates)
    else:
        return("Problem optimizing")
